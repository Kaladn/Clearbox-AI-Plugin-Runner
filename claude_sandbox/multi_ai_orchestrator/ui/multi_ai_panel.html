⬡ Multi-AI Orchestrator — Help

Modes

ModeBehavior
HUBYou dispatch. No auto-wiring between nodes.
CHAINEach node can wire its output to the next node's input.

Slot Types

SlotBehavior
LLMCalls the selected AI provider. Uses vscode.lm, extension export, or local HTTP.
HUMAN ✎You type the assistant response. Chain always pauses here. Never auto-fires.

Provider Tiers

TierSource
vscode.lmAny VSCode AI extension that registers via vscode.lm API (Copilot, etc.)
Extension exportConfigure in settings: clearboxMultiAi.knownExtensions
Local HTTPConfigure in settings: clearboxMultiAi.localEndpoints — localhost/LAN only

Chain Fire Modes

PREVIEW_FIRST (default) — injects text into next node's input, waits for you to confirm before firing.

AUTO — fires next node immediately on completion. HUMAN nodes always override to PREVIEW_FIRST.

Session Storage

Every turn written to disk immediately on completion.
Location: ~/.clearbox/multi_ai/sessions/{session_id}/turns.jsonl
Append-only. Never rewritten. Never summarized. Full content always.

session_notes.md — never auto-written. Yours only. Opens in a panel beside this one.

Export Formats

JSONL — raw turns file. One JSON record per line. wolf_engine compatible.

Markdown — handoff log with node labels, sources, and timestamps. Human readable.
