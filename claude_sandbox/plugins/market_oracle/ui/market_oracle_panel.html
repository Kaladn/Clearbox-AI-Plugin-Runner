Market Oracle — Help & API Schema

What This Plugin Does

Fetches historical market data via yfinance, builds 6-1-6 temporal capsules (6 prev + 1 anchor + 6 next trading days), runs bidirectional causal validation, detects behavioral regimes from rolling consistency bands, stores results in SQLite.

No ML models. No embeddings. Count-based deterministic analysis.

Regime Types

ORDERED consistency > 0.60 — predictable, institutional behavior. Risk: LOW
MODERATE 0.40–0.60 — mixed signals, transitional. Risk: MEDIUM
VOLATILE 0.25–0.40 — weak causal structure. Risk: HIGH
CHAOTIC ≤ 0.25 — no causal structure. Risk: EXTREME

Regime Change Severity

CRITICAL — delta > 0.20
MAJOR — delta 0.10–0.20
MINOR — delta < 0.10

Field Reference

Pattern Break Threshold — % price change that counts as a break. Default: 5.0. Lower = more sensitive.

Capsule Window (N-1-N) — positions on each side of anchor. Default: 6 (13-point window). Sets NCV-73 at N=6.

Regime Rolling Window — days used to compute rolling avg consistency before classifying a regime. Default: 10.

Min Regime Duration — minimum consecutive days to call it a real regime. Shorter segments are absorbed by neighbor. Default: 5.

High Consistency Threshold — above this = causal chain. Default: 0.70.

Low Consistency Threshold — below this = anomaly. Default: 0.30.

Min Chain Length — min days above high threshold to count as a causal chain. Default: 5.

API Endpoints

POST /api/market_oracle/analyze
Body: {"tickers":["MSFT"],"config":{"period":"2y",...}}
Returns: full analysis result or comparison.

POST /api/market_oracle/history
Body: {"ticker":"MSFT","limit":20}

POST /api/market_oracle/regimes
Body: {"analysis_id":42}

GET /api/market_oracle/config — default config + regime band schema

GET /api/market_oracle/help — full machine-readable API schema (this content, as JSON)

Output Paths

All file output is saved to absolute paths under ~/.clearbox/market_oracle/

Reports: ~/.clearbox/market_oracle/reports/
Charts: ~/.clearbox/market_oracle/charts/
Database: ~/.clearbox/market_oracle/market_oracle.db

Storage Schema

SQLite tables: analyses, regimes, regime_changes, trading_days. WAL mode. Foreign keys enforced. Safe to delete and rebuild — all data regenerable from yfinance.
